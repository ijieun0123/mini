### ✨ **`mini` 사이트 리뉴얼 프로젝트**

반응형 디자인과 향상된 사용자 경험을 위한 리뉴얼 프로젝트입니다.<br/>
https://ijieun0123.github.io/mini/

---

### 🚀 **프로젝트 동기**

기존 사이트의 오래된 디자인을 개선하고, 사용자 친화적인 인터페이스를 제공하기 위해 React를 활용한 리뉴얼을 진행했습니다. 특히, 모바일에 최적화된 반응형 웹 디자인과 동적인 스크롤 애니메이션 효과에 중점을 두었습니다.

---

### 🛠️ **기술 스택**

-   **언어:** `TypeScript`
-   **프레임워크:** `React.js`
-   **번들러:** `Vite`
-   **스타일링:** `Styled-Components`, `SCSS`
-   **패키지 매니저:** `Yarn`
-   **애니메이션:** `AOS`
-   **슬라이더:** `Swiper.js`

---

### 🛠 핵심 기술 구현 상세

<details>
<summary><b>1. TypeScript를 통한 "Type-Safe UI" 구현</b></summary>
<br>

-   **엄격한 데이터 관리:** 모든 UI 데이터와 Props에 Interface를 적용하여 데이터 구조를 엄격하게 관리했습니다.
-   **오류 차단 및 생산성 향상:** 협업 시 발생할 수 있는 데이터 타입 불일치 오류를 사전에 차단하고, 자동 완성을 통해 개발 생산성을 높였습니다.
</details>

<details>
<summary><b>2. Styled-components 기반의 컴포넌트 주도 설계</b></summary>
<br>

-   **Props를 활용한 동적 스타일링:** 동일한 버튼 컴포넌트라도 전달되는 Props(`color`)에 따라 UI가 즉각 변경되도록 설계하여, 선언적 UI 관리를 구현했습니다.
-   **컴포넌트 단위 스타일 캡슐화:** 각 컴포넌트 내부에서 스타일을 정의하여 외부와의 충돌을 방지하고 코드의 독립성을 확보했습니다.
-   **재사용성 극대화:** 독립적인 부품 단위로 요소를 분리하여, 수정 사항 발생 시 해당 컴포넌트만 수정하면 전체 프로젝트에 반영되는 효율적인 구조를 구축했습니다.
</details>

<details>
<summary><b>3. "Pixel-Perfect"를 지향하는 정밀한 UI 구현</b></summary>
<br>

-   **수치 기반의 완벽한 재현:** 피그마 시안의 절대 수치를 분석하여 브라우저 환경에 맞는 상대 비율(%)로 정밀하게 환산, 어떤 해상도에서도 디자인 의도가 유지되도록 구현했습니다.
-   **유연하고 견고한 반응형 설계:** 주요 브레이크 포인트(768px, 1024px, 1440px)별로 폰트 크기, 자간, 여백을 재설정하여 해상도에 상관없이 최적의 시각적 밸런스를 유지했습니다.
-   **전략적 콘텐츠 노출 관리:** `hide_on_mobile` 클래스 등을 활용해 기기별 최적화된 콘텐츠만 노출함으로써 시각적 밸런스를 최적화했습니다.
</details>

<details>
<summary><b>4. 사용자 경험을 고려한 인터랙션 및 라이브러리 최적화</b></summary>
<br>

-   **스크롤 간섭 방지 및 정밀 제어:** `EditionSlider`에 `forceToAxis: true` 옵션을 적용하여 가로 슬라이드와 세로 스크롤 간의 간섭을 차단하고 조작 편의성을 높였습니다.
-   **해상도별 가변 레이아웃:** `NewsSlider`에 `slidesPerView: "auto"` 설정을 적용하여, 화면 크기에 따라 카드가 유연하게 배치되도록 설계함으로써 콘텐츠 절단 현상을 최소화했습니다.
-   **브랜드 무드 최적화:** `Swiper.js`의 전환 속도를 세밀하게 조정하고, `AOS.js`의 지연 시간과 지속 시간을 시선 흐름에 맞춰 설계하여 리듬감 있는 UI를 구현했습니다.
</details>

<details>
<summary><b>5. SCSS 시스템 설계 및 모듈화 작업</b></summary>
<br>

-   **디자인 시스템 변수화:** 컬러, 폰트 규격 등을 `_variables.scss`로 정의하여 일관된 디자인 시스템을 구축했습니다.
-   **스타일 패턴 재사용:** 반복되는 CSS 속성을 `_mixins.scss`로 모듈화하여 코드 중복을 최소화했습니다.
-   **반응형 가이드 및 레이아웃 설계:** `_common.scss`에서 피그마 시안 기반의 정밀한 해상도별 비율 계산을 통해 .container를 규격화했습니다. 또한 텍스트 위계에 따른 반응형 스타일을 공통 클래스로 관리하여 스타일 파편화를 방지했습니다.
-   **구조적 스타일 관리:** 스타일 시트를 `reset`, `variables`, `mixins`, `common`으로 분리하여 스타일 충돌을 방지하고 대규모 프로젝트에서도 유지보수가 용이한 구조를 지향합니다.
</details>

### 🔍 트러블슈팅

<details>
<summary><b>1. 해상도별 배치 전략에 따른 마크업 구조 리팩토링</b></summary>
<br>

-   **Problem:** `Digital Experience` 섹션에서 모바일(수직 나열)과 데스크탑(복잡한 좌표 기반 배치)의 구조 차이가 커 일반적인 CSS 수정만으로는 대응이 어려웠습니다.
-   **Solution:** 디자인 재현율을 높이기 위해 배치 전략을 이원화했습니다. 모바일은 기본 문서 흐름을 따르고, 데스크탑은 절대 좌표(`position: absolute`)를 활용하여 시안과 오차 없이 배치했습니다. 이를 위해 마크업 계층 구조를 전면 재설계했습니다.
-   **Result:** 데스크탑에서 복잡한 시각적 배치를 재현함과 동시에 모바일에서도 안정적인 반응형 레이아웃을 구현했습니다.
</details>

<details>
<summary><b>2. 고정 오브젝트와 인터랙션 영역의 레이어 분리 구현</b></summary>
<br>

-   **Problem:** `MINI NEWS` 섹션에서 거대한 자동차 배경 이미지를 고정하면서도 슬라이더 인터랙션이 방해받지 않도록 설계해야 했습니다. 이미지의 크기 때문에 발생하는 레이아웃 틀어짐과 클릭 간섭이 주요 난관이었습니다.
-   **Solution:**
    -   **중앙 기준 좌표 설계:** 자동차 이미지에 `position: absolute`를 설정하고, `right: 50%`와 `margin-right` 값을 조합하여 화면 중앙축을 기준으로 위치를 고정했습니다. 이를 통해 해상도 변화에도 자동차가 항상 제자리를 유지하게 했습니다.
    -   **독립 인터랙션 레이어:** 배경과 슬라이더 레이어를 분리하여 카드 슬라이딩 시 배경 요소와 물리적 충돌이 없도록 구현했습니다.
-   **Result:** 브랜드 이미지의 시각적 임팩트를 유지하면서 사용자가 방해 없이 슬라이더를 조작할 수 있는 인터페이스를 완성했습니다.
</details>
